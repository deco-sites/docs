## Conteudo do CMS

O CMS da deco gerencia os conteudos dos blocos da deco (framework). Estes blocos são salvos em JSONs dentro da pasta `/.deco/blocks/` do projeto do site. O nome do arquivo JSON é o identificador do bloco, que possui este formato `/.deco/blocks/{id-do-bloco}.json` onde este id é escrito por um usuário.

## Identificando id do bloco de uma página deco

Dado que tenho acesso ao uma página de um site deco, basta navegar para a página, no navegador, e em seguida acessar
a variável da `window` desta forma: `window.LIVE.page.id`. Com o id do bloco de página, basta acessar o arquivo `/.deco/blocks/{id-da-pagina}.json`

## Identificando quais sections são utilizadas na página deco

Basta acessar a página deco através do navegador, as tags HTML `section` filhas do body que possuem a propriedade `data-manifest-key`.

Algumas sections são especiais, como a section `website/sections/Rendering/Lazy.tsx` que encapsula outra section, neste caso observe o data-manifest-key da tag HTML section filha imediata.
